## Application Building Context

Read the following files in order before implementing or making any architectural decision:

1. `DOCS/Project-Overview.md` — product definition, scope, user flow, layers, and success criteria
2. `DOCS/architecture.md` — system diagram, data flow, file structure, and where to add new code
3. `DOCS/Tech-stack.md` — technology choices and rationale
4. `DOCS/roadmap.md` — day-by-day build plan and verification checklists
5. `DOCS/progress-tracker.md` — current phase, completed work, open questions, and next steps
6. `DOCS/Current-issues.md` — active bugs, blockers, and known broken behavior
7. `DOCS/Startup.md` — how to run the project locally (commands, env setup, prerequisites)
8. `DOCS/Notes.md` — loose decisions, open questions, and context that doesn't fit elsewhere

Update `DOCS/progress-tracker.md` after each meaningful implementation change.

If a bug or blocker is discovered, log it in `DOCS/Current-issues.md` before continuing.

When working on a specific feature, also read the relevant spec in `DOCS/Feature-specs/` before starting.

If implementation changes the architecture or scope documented in the context files, update the relevant file before continuing.
